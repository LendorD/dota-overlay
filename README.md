## Компоненты
- `overlay` — основной оверлей (контр-пики, лучший пик, GSI, парсер логов).
- `dotaplus` — компактное окно со статистикой из GSI.
- `launcher` — запускает оба окна одной командой.

## Быстрый старт

1) Установи Go.
2) Запусти лаунчер:

```bash
go run ./cmd/launcher
```

### Запуск по отдельности

```bash
go run ./cmd/overlay
go run ./cmd/dotaplus
```

## Управление

### Overlay
- `F12` — Lock/Unlock (в unlocked режиме окно кликабельно и перетаскивается).
- ЛКМ (клик) — переключение контр-пиков по найденным героям.
- `+` / `-` — масштаб окна.

### Dota Plus
- Перетаскивание — за верхнюю панель.
- Растягивание — за правый нижний угол.

## GSI (Game State Integration)

1) Скопируй конфиг `gsi/gamestate_integration_overlay.cfg` в папку Dota:
`C:\Program Files (x86)\Steam\steamapps\common\dota 2 beta\game\dota\cfg\`
2) Перезапусти игру.

GSI шлёт события на локальный сервер `http://127.0.0.1:3001/`, который запускается вместе с `overlay`.

## OpenDota API

Если есть ключ, можно передать через переменную окружения:

```powershell
$env:OPENDOTA_API_KEY="YOUR_KEY"
```

## Логи
- `gsi_log.txt` — сырые GSI события.
- `log_dota.txt` — логи из `console.log`.

## Примечания
- В некоторых режимах (боты/локалки/некоторые лобби) пики могут не приходить.
- Для полноценного списка героев приложение загружает `/heroes` из OpenDota при старте.
- `dotaplus` получает данные через `overlay`, поэтому `overlay` должен быть запущен.
